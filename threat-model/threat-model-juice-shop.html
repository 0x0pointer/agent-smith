Shostack's 4 Questions

Q1 — What are we working on?

An e-commerce juice shop SPA with REST API, SQLite backend, JWT/OAuth auth, file storage, and admin functionality — running over plain HTTP with no WAF or rate limiting.

Q2 — What can go wrong?

SQLi admin bypass, credential brute-force, confidential file download, cleartext credential sniffing, XSS via missing CSP, CORS-enabled cross-origin data theft, cart price manipulation.

Q3 — What are we going to do about it?

See Risk Register (Section 7). Three-horizon mitigations for every High/Critical threat: immediate hotfix, next release, and architectural hardening.

Q4 — Did we do a good job?

See Retrospective (Section 8). All 4 quality criteria met: coverage, depth, actionability, and business alignment.

1. Objectives & Scope

Business Objectives

  • Protect customer PII and payment data: User accounts contain email addresses, passwords (hashed), order history, and payment card details.
  • Maintain e-commerce availability: Shopping cart, checkout, and product catalog are revenue-generating — downtime or data corruption has direct financial impact.
  • Protect business-confidential documents: M&A acquisition plans (acquisitions.md) are accessible via the FTP directory; unauthorized disclosure could enable insider trading or competitive intelligence.
  • Preserve user account integrity: Admin-level access allows price manipulation, user account management, and order data access.
  • Comply with PII/payment regulations: GDPR applies to EU user data; PCI-DSS applies to any cardholder data in scope.

Cart Price Manipulation

Product prices are returned via the API at page load but may not be re-validated at checkout. An attacker intercepting the checkout payload can replace the price field with 0 or a negative value to purchase items at manipulated prices. This is a classic server-side validation bypass where the UI enforces business rules but the API does not.

Admin Role Assignment via API

The admin panel is protected only by an Angular route guard (client-side). If the underlying REST API endpoints (e.g., /api/Users/:id) do not enforce role checks server-side, a regular user can call admin API endpoints directly with their own JWT token and receive admin-level data or perform admin operations.

Coupon Double-Spend (Race Condition)

Coupon code and wallet credit redemption — if tracked with a simple used boolean and not an atomic database transaction — is susceptible to a concurrent-request race condition. Sending two simultaneous redemption requests may apply the same coupon twice before the used flag is set to true.

Deluxe Membership Bypass

Deluxe pricing (e.g., Apple Juice: deluxePrice 0.99 vs regular 1.99) is gated on membership tier. If the tier check happens via a JWT claim that is not validated server-side on each cart operation, a regular user can craft a request with a manipulated tier claim and receive deluxe pricing without paying for membership.

Registration / Login User Enumeration

If login error messages differ between "user not found" and "wrong password", attackers can enumerate valid email addresses before running credential stuffing — reducing the search space significantly.

Password Reset via Security Questions

Security question-based password reset is susceptible to brute-force (limited answer space for personal questions) and social engineering. No rate limiting observed on login applies equally here.

7. Risk Register & Mitigations

Activity 2: Critical Assessment

Sorted by Risk (Critical first), then Business Impact. Mitigations given in three horizons: Immediate / Short-term / Long-term.

Critical

Critical   T-01 — No TLS/HTTPS

Business Impact: Every credential, JWT token, PII record, and Google OAuth code is readable in cleartext. A passive observer on any network segment can harvest accounts at scale without active exploitation.
ImmediateEnable HTTPS with TLS 1.2+ certificate. Add Strict-Transport-Security: max-age=31536000; includeSubDomains. Redirect all HTTP to HTTPS in Express.
Short-termAutomate certificate renewal (Let's Encrypt / ACM). Enforce TLS in reverse proxy config. Update Google OAuth redirect URI to HTTPS in Cloud Console.
Long-termEnable HSTS preloading. Implement PKCE for OAuth flow. Move to RS256 JWT signing (asymmetric) so signing key is never transmitted.

Critical   T-02 — SQL Injection on Login

Business Impact: A single unauthenticated HTTP request with ' OR 1=1-- grants admin access to the entire application — user PII, orders, payment details, and all admin functions. Zero prerequisites required.
ImmediateReplace string interpolation in login SQL with parameterized queries: WHERE email = ? AND password = ? using prepared statements.
Short-termAudit all SQL queries — migrate to Knex.js query builder or Sequelize/Prisma ORM that prevents raw SQL construction.
Long-termAdd Semgrep rule javascript.lang.security.audit.sqli to CI pipeline. Enable SQLi detection in WAF rules. Add automated SQLi tests to integration suite.

High

High   T-03 — No Rate Limiting on Login

Business Impact: Unlimited login attempts enable credential stuffing from breach databases at thousands of requests per second. All user accounts are vulnerable to takeover without any technical barrier.
ImmediateAdd express-rate-limit middleware: max 5 attempts per IP per minute on /rest/user/login with exponential backoff after 3 failures.
Short-termAdd account-level lockout (10 failed attempts triggers 15-min lock + email notification). Implement CAPTCHA after 3 failures.
Long-termIntegrate threat intelligence feed to block known credential-stuffing IP ranges. Add anomaly detection on login velocity per account.

High   T-04 — Unauthenticated Admin Config Endpoint

Business Impact: Exposes Google OAuth clientId, server configuration, all product data, and internal feature flags to any unauthenticated user — enables targeted OAuth abuse and lowers the barrier for all other attacks by revealing the application's internals.
ImmediateAdd JWT authentication middleware to the /rest/admin/* route group. Verify isAdmin JWT claim before responding.
Short-termAudit all /rest/admin/* routes for missing auth middleware. Add integration tests asserting 401 for unauthenticated requests to all admin routes.
Long-termImplement RBAC middleware applied at the router level so new routes inherit protection by default rather than requiring per-route decoration.

High   T-05 — FTP Directory Listing and Confidential File Access

Business Impact: Confidential M&A acquisition plans are freely downloadable. Backup files in the directory may contain database dumps with PII and hashed passwords. Regulatory exposure (GDPR) for any EU user data in accessible files.
ImmediateRemove acquisitions.md and all backup/database files from the web-accessible /ftp/ directory immediately. Disable directory listing in Express static middleware.
Short-termRequire authentication for all /ftp/* requests. Audit all files currently in the directory for sensitivity.
Long-termMove sensitive documents to a private object store (S3 with bucket policies, or filesystem outside web root). Serve downloads via signed URLs with expiry and access logging.

High   T-06 — SQL Injection on Product Search

Business Impact: A UNION-based injection on /rest/products/search?q= can dump the entire Users table including email addresses, hashed passwords, and order history — full PII breach affecting all users.
ImmediateParameterize the search query: SELECT * FROM Products WHERE name LIKE ? with ['%' + term + '%']. Never concatenate user input into SQL strings.
Short-termSame ORM migration as T-02. Add automated SQLi test for the search endpoint to the CI integration suite.
Long-termSame SAST + WAF as T-02.

High   T-07 — Missing Content Security Policy (XSS Enabler)

Business Impact: Any stored or reflected XSS payload (via product reviews, usernames, chatbot responses) executes without restriction. Attackers can exfiltrate JWT tokens from localStorage, perform admin actions as the victim, and install persistent payloads visible to all future visitors.
ImmediateAdd strict CSP header: default-src 'self'; script-src 'self'; object-src 'none'; base-uri 'self'. This blocks inline scripts and external script sources immediately.
Short-termAudit Angular template bindings for [innerHTML] — replace with DomSanitizer. Move JWT from localStorage to httpOnly cookies to prevent exfiltration even if XSS occurs.
Long-term">Add SRI (Subresource Integrity) for CDN scripts. Enable CSP reporting endpoint to detect violations in production.

High   T-08 — Potential Weak JWT Secret

Business Impact: If the JWT signing secret is a default or weak value, all tokens can be forged offline — attacker creates an admin-role JWT and gains full admin access without any credentials.
ImmediateRotate JWT secret to a cryptographically random 256-bit value stored in an environment variable or secrets manager — never in source code.
Short-term">Implement JWT expiry (15-minute TTL) with refresh token rotation. Validate iss, aud, and exp claims on every request.
Long-term">Migrate to RS256 (asymmetric signing) so the private signing key is never distributed or included in application config.

Medium

Medium   T-10 — Wildcard CORS Policy

Business Impact: Any malicious website can silently read full API responses from the victim's browser session, amplifying every other information disclosure vulnerability into a cross-origin exfiltration primitive.
ImmediateReplace Access-Control-Allow-Origin: * with an explicit allowlist: http://localhost:3000 for dev, specific production domain for prod.
Short-term">Audit all Express CORS middleware configuration. For credentialed endpoints, set credentials: true with specific origin (wildcard + credentials is rejected by browsers anyway).
Long-term">Add automated header assertion tests to CI that fail if wildcard CORS is re-introduced.

Medium   T-11 — Verbose Error Messages / Stack Traces

Business Impact: Full stack traces reveal internal file paths, framework versions, and dependency layout — reducing reconnaissance effort for targeted exploitation of known CVEs.
ImmediateAdd global Express error handler returning generic message: res.status(500).json({error: 'Internal server error'}).
Short-term">Set NODE_ENV=production — Express suppresses stack traces automatically. Add error monitoring (Sentry) so errors are still logged server-side.
Long-term">Implement structured logging with correlation IDs so error details are queryable internally but never exposed to clients.

Medium   T-12 — jQuery 2.2.4 (CVE-2015-9251, CVE-2019-11358)

Business Impact: Known CVEs in jQuery 2.2.4 enable XSS via prototype pollution and unsafe HTML rendering. Exploitable wherever user-supplied data is passed to jQuery DOM manipulation methods.
ImmediateUpgrade jQuery to 3.7.x. Audit all .html(), .append(), and .parseHTML() calls for user-supplied data.
Short-term">Run npm audit and resolve all high/critical dependency vulnerabilities. Add lockfile audit to CI (npm audit --audit-level=high).
Long-term">Adopt Dependabot or Renovate for automated dependency update PRs. Replace jQuery DOM manipulation with Angular's built-in safe templating.

  1. Confirm SQLi on /rest/products/search — not fully validated in this engagement. Run sqlmap with --ignore-code=401 on the search endpoint.
  2. Test payment flow for price manipulation — add to next sprint's manual test scope. Focus on intercepting the checkout payload and modifying the price field.
